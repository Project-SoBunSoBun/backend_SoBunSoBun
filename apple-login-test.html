🍎 Apple 로그인 테스트

로컬 환경에서 Apple Sign In 백엔드 API를 테스트합니다.

⚙️ 서버 설정

1Apple JS SDK로 로그인 (권장)

Apple 공식 JS SDK를 사용하여 로그인합니다.
⚠️ Apple JS SDK는 HTTPS에서만 작동합니다. 로컬에서는 방법 2 또는 3을 사용하세요.

2수동으로 Apple 인증 URL 열기 (로컬 테스트용)

Apple 인증 페이지를 직접 열어서 authorization_code를 받습니다.
콜백은 https://www.sobunsobun.store/auth/callback/apple로 가므로,
프로덕션 서버 응답에서 code를 복사하여 아래 방법 3에서 사용하세요.

💡 플로우:
1. 버튼 클릭 → Apple 로그인 페이지 열림
2. Apple ID로 로그인
3. 콜백 URL로 code와 id_token이 POST됨
4. 프로덕션 서버 응답 확인 또는 code를 복사하여 아래에서 직접 테스트

3직접 API 호출 (가장 실용적)

iOS 앱 시뮬레이터 또는 Apple 인증 페이지에서 받은 값을 직접 입력하여 테스트합니다.

4회원가입 완료 (loginToken으로 JWT 발급)

위 단계에서 받은 loginToken으로 회원가입을 완료합니다.

⚠️ 로컬 테스트 시 주의사항

        
    
            
  • Apple JS SDK는 HTTPS에서만 작동합니다. 로컬에서는 방법 2, 3을 사용하세요.
    • 
            
  • Authorization Code는 1회용이며, 5분 내에 사용해야 합니다.
    • 
            
  • id_token만으로도 검증 가능합니다 (iOS 앱에서 identityToken 전달 시).
    • 
            
  • 이메일은 최초 로그인 시에만 Apple이 제공합니다. 재로그인 시 이메일이 없을 수 있습니다.
    • 
            
  • iOS 시뮬레이터에서 Apple 로그인 → Xcode 콘솔에서 identityToken 출력 → 위 id_token에 붙여넣기
    • 
        

    


    

        
📱 iOS 앱에서 테스트하는 방법

        
    
            
  1. Xcode에서 iOS 앱 실행 (시뮬레이터 또는 실기기)
    2. 
            
  2. Apple 로그인 버튼 탭
    3. 
            
  3. 로그인 완료 시 ASAuthorization 콜백에서:
    4. 
            
  4. credential.identityToken → UTF-8 문자열로 변환
    5. 
            
  5. credential.authorizationCode → UTF-8 문자열로 변환
    6. 
            
  6. 해당 값을 POST /auth/verify/apple-token에 JSON으로 전송:
    
                { "code": "authorizationCode값", "idToken": "identityToken값" }
            
    7. 
            
  7. 응답의 loginToken으로 POST /auth/complete-signup 호출
    8. 
        

    


    

        
🧪 cURL 테스트 명령어

        
# 1단계: Apple 토큰 검증
curl -X POST http://localhost:8081/auth/verify/apple-token \
  -H "Content-Type: application/json" \
  -d '{
    "code": "YOUR_AUTHORIZATION_CODE",
    "idToken": "YOUR_ID_TOKEN"
  }'

# 2단계: 회원가입 완료
curl -X POST http://localhost:8081/auth/complete-signup \
  -H "Content-Type: application/json" \
  -d '{
    "loginToken": "RECEIVED_LOGIN_TOKEN",
    "serviceTermsAgreed": true,
    "privacyPolicyAgreed": true,
    "marketingOptionalAgreed": false
  }'
